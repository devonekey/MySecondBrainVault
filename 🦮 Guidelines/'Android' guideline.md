## 😋 개인 취향·생각

- 최신 기술 트렌드를 따름
- 복잡한 기능을 가진 프로젝트일수록 설계가 짜임새 있어야 하며, 테스트 코드가 있어야 코드를 신뢰할 수 있음
- `targetSdk` 버전은 최신 버전으로 사용하되, `minSdk`까지의 하위 버전에도 기능이 똑같이 적용되거나 대안이 존재해야 함
- `Google` 및 `Jetbrains` 등에서 제공하는 공식적인 라이브러리 사용을 선호함
- SAA(Single Activity Architecture)까지는 아니더라도 최소한의 `Activity`를 써야 한다고 생각함
- `Activity`는 이벤트를 유발하는 대상으로 간주하며, `Activity`로부터 UI를 분리해야 한다고 생각함
- 하나의 파일(테스트 및 리소스 파일 등은 제외)에 작성된 코드는 가급적 500줄 내로 작성돼야 한다고 생각함
- 축약형은 쓰지 않되 간결한 네이밍을 선호함
- 메서드 체이닝 방식을 선호함
- 기능을 직접 구현하는 것은 최소한으로 이용하고, 이미 검증된 기능을 최대한 이용하는 것을 선호함
- 'Coroutine', 'Flow'를 반드시 적용해야 한다고 생각함
